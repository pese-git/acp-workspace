## Проблема

Самописный DI-контейнер (~440 строк суммарно) содержал несколько принципиальных изъянов:

| Проблема | Проявление |
|---|---|
| Не делает DI — только хранит объекты | Все сервисы создаются вручную в bootstrapper |
| SCOPED = SINGLETON (TODO) | Ложный контракт, нет per-request изоляции |
| `dispose()` синхронный | Async-ресурсы (WebSocket) не закрываются корректно |
| Циклическая зависимость через мутацию приватного поля | `coordinator._permission_handler = permission_handler` |
| `# type: ignore[call-top-callable]` | Несовместимость типов в `Registration.create()` |

**Решение:** заменить на [`dishka`](https://github.com/reagento/dishka) — async-first DI для Python с правильными scopes и lifecycle через `AsyncContextManager`.

## Шаг 2 — Разорвать циклическую зависимость

### Реализованное решение: односторонняя зависимость

Исходный цикл возникал из-за того, что `PermissionHandler` после создания прокидывался в `SessionCoordinator` через мутацию приватного поля, а внутри мог обратно ссылаться на координатор. Цикл разорван через `PermissionCallbackRegistry` — разделяемый объект состояния, к которому обращаются обе стороны без прямой ссылки друг на друга:

```python
# БЫЛО: мутация после создания
coordinator._permission_handler = permission_handler  # нарушает DI

# СТАЛО: SessionCoordinator получает PermissionHandler через конструктор,
# PermissionHandler не знает о SessionCoordinator вообще.
# Обратная связь (resolve/cancel) идёт через PermissionCallbackRegistry.

class SessionCoordinator:
    def __init__(self, transport, session_repo, permission_handler: PermissionHandler | None):
        self._permission_handler = permission_handler  # прямая ссылка, но односторонняя

class PermissionHandler:
    def __init__(self, event_bus: EventBus, permission_callback_registry: PermissionCallbackRegistry):
        # Нет ссылки на SessionCoordinator — цикл разорван
```

`PermissionCallbackRegistry` — это разделяемый реестр, который инжектируется и в `ACPTransportService`, и в `PermissionHandler`, и позволяет им координироваться без прямой зависимости.

### Альтернатива: полный EventBus-декаплинг

Более строгий вариант — убрать прямую ссылку `SessionCoordinator → PermissionHandler` полностью, заменив на обмен событиями через шину:

```python
class SessionCoordinator:
    def __init__(self, transport, session_repo, event_bus: EventBus):
        self._event_bus = event_bus
        # Нет прямой ссылки на PermissionHandler

class PermissionHandler:
    def __init__(self, event_bus: EventBus, permission_callback_registry: PermissionCallbackRegistry):
        self._event_bus = event_bus
        # Нет прямой ссылки на SessionCoordinator
```

`SessionCoordinator` публикует `PermissionRequestEvent` → `PermissionHandler` слушает, показывает UI, публикует `PermissionResponseEvent` → `SessionCoordinator` слушает и продолжает выполнение.

**Что даёт:**
- Полная тестовая изоляция: `SessionCoordinator` тестируется без `PermissionHandler` вообще
- Сменяемость стратегии без изменения координатора (авто-approve, CLI-prompt, удалённый approval)
- Возможность нескольких обработчиков для разных типов операций

**Почему не реализовано сейчас:**
`SessionCoordinator` обращается к `PermissionHandler` за синхронным состоянием через `get_request_manager()` — для resolve/cancel активных запросов. Переход на EventBus потребует асинхронного request-response с correlation ID и `asyncio.Future` на каждый запрос, что значительно усложняет код без реального выигрыша для одного TUI-клиента с единственной стратегией permission.

Реализовать имеет смысл, если появится требование: несколько стратегий в рантайме, кросс-процессный permission flow или полная изоляция координатора в тестах.

## Шаг 3 — Создать `AppProvider`

Файл `src/codelab/client/infrastructure/app_provider.py` (~194 строки):

```python
"""dishka Provider — декларативная конфигурация зависимостей клиентского приложения."""

from __future__ import annotations

from collections.abc import AsyncIterator
from pathlib import Path

import structlog
from dishka import Provider, Scope, provide

from codelab.client.application.permission_handler import PermissionHandler
from codelab.client.application.session_coordinator import SessionCoordinator
from codelab.client.infrastructure.events.bus import EventBus
from codelab.client.infrastructure.events.permission_events import PermissionCallbackRegistry
from codelab.client.infrastructure.handlers.file_system_handler import FileSystemHandler
from codelab.client.infrastructure.handlers.terminal_handler import TerminalHandler
from codelab.client.infrastructure.repositories import InMemorySessionRepository
from codelab.client.infrastructure.services.acp_transport_service import ACPTransportService
from codelab.client.infrastructure.services.file_system_executor import FileSystemExecutor
from codelab.client.infrastructure.services.terminal_executor import TerminalExecutor
from codelab.client.presentation.chat_view_model import ChatViewModel
from codelab.client.presentation.file_viewer_view_model import FileViewerViewModel
from codelab.client.presentation.filesystem_view_model import FileSystemViewModel
from codelab.client.presentation.permission_view_model import PermissionViewModel
from codelab.client.presentation.plan_view_model import PlanViewModel
from codelab.client.presentation.session_view_model import SessionViewModel
from codelab.client.presentation.terminal_log_view_model import TerminalLogViewModel
from codelab.client.presentation.terminal_view_model import TerminalViewModel
from codelab.client.presentation.ui_view_model import UIViewModel

logger = structlog.get_logger("app_provider")


class AppProvider(Provider):
    scope = Scope.APP

    def __init__(self, host: str, port: int, cwd: str, history_dir: str | None = None):
        super().__init__()
        self._host = host
        self._port = port
        self._cwd = cwd
        self._history_dir = history_dir

    # --- Инфраструктурные сервисы ---

    @provide
    def event_bus(self) -> EventBus:
        return EventBus()

    @provide
    def permission_callback_registry(self) -> PermissionCallbackRegistry:
        # Разделяемый реестр — инжектируется в transport и permission_handler
        return PermissionCallbackRegistry()

    @provide
    async def transport(
        self,
        event_bus: EventBus,
        permission_callback_registry: PermissionCallbackRegistry,
    ) -> AsyncIterator[ACPTransportService]:
        service = ACPTransportService(
            host=self._host,
            port=self._port,
            event_bus=event_bus,
            permission_callback_registry=permission_callback_registry,
        )
        yield service
        await service.disconnect()
        logger.debug("transport_closed")

    @provide
    def session_repository(self) -> InMemorySessionRepository:
        return InMemorySessionRepository()

    @provide
    def fs_executor(self) -> FileSystemExecutor:
        return FileSystemExecutor(base_path=Path(self._cwd))

    @provide
    async def terminal_executor(self) -> AsyncIterator[TerminalExecutor]:
        executor = TerminalExecutor()
        yield executor
        await executor.cleanup_all()
        logger.debug("terminal_executor_cleaned_up")

    # --- Обработчики ---

    @provide
    def fs_handler(self, executor: FileSystemExecutor) -> FileSystemHandler:
        return FileSystemHandler(executor=executor)

    @provide
    def terminal_handler(self, executor: TerminalExecutor) -> TerminalHandler:
        return TerminalHandler(executor)

    # --- Application слой ---

    @provide
    def permission_handler(
        self,
        event_bus: EventBus,
        permission_callback_registry: PermissionCallbackRegistry,
    ) -> PermissionHandler:
        return PermissionHandler(
            event_bus=event_bus,
            permission_callback_registry=permission_callback_registry,
            logger=logger,
        )

    @provide
    def session_coordinator(
        self,
        transport: ACPTransportService,
        session_repo: InMemorySessionRepository,
        permission_handler: PermissionHandler,
    ) -> SessionCoordinator:
        return SessionCoordinator(
            transport=transport,
            session_repo=session_repo,
            permission_handler=permission_handler,
        )

    # --- ViewModels ---

    @provide
    def ui_view_model(self, event_bus: EventBus) -> UIViewModel:
        return UIViewModel(event_bus=event_bus, logger=logger)

    @provide
    def session_view_model(
        self, coordinator: SessionCoordinator, event_bus: EventBus
    ) -> SessionViewModel:
        return SessionViewModel(coordinator=coordinator, event_bus=event_bus, logger=logger)

    @provide
    def plan_view_model(self, event_bus: EventBus) -> PlanViewModel:
        return PlanViewModel(event_bus=event_bus, logger=logger)

    @provide
    def chat_view_model(
        self,
        coordinator: SessionCoordinator,
        event_bus: EventBus,
        fs_executor: FileSystemExecutor,
        terminal_executor: TerminalExecutor,
        plan_vm: PlanViewModel,
    ) -> ChatViewModel:
        return ChatViewModel(
            coordinator=coordinator,
            event_bus=event_bus,
            logger=logger,
            history_dir=self._history_dir,
            fs_executor=fs_executor,
            terminal_executor=terminal_executor,
            plan_vm=plan_vm,
        )

    @provide
    def terminal_view_model(self, event_bus: EventBus) -> TerminalViewModel:
        return TerminalViewModel(event_bus=event_bus, logger=logger)

    @provide
    def filesystem_view_model(self, event_bus: EventBus) -> FileSystemViewModel:
        return FileSystemViewModel(event_bus=event_bus, logger=logger)

    @provide
    def file_viewer_view_model(self, event_bus: EventBus) -> FileViewerViewModel:
        return FileViewerViewModel(event_bus=event_bus, logger=logger)

    @provide
    def permission_view_model(self, event_bus: EventBus) -> PermissionViewModel:
        return PermissionViewModel(event_bus=event_bus, logger=logger)

    @provide
    def terminal_log_view_model(self, event_bus: EventBus) -> TerminalLogViewModel:
        return TerminalLogViewModel(event_bus=event_bus, logger=logger)
```

## Шаг 4 — Обновить точку входа приложения

Контейнер создаётся и управляется в `_run_tui_app_async` через `async with` — это гарантирует cleanup даже если `on_mount` упадёт на середине разрешения ViewModels (например, когда WebSocket уже открыт, а следующий `container.get()` бросает исключение). `ACPClientApp` получает готовый контейнер и не владеет его lifecycle.

```python
# src/codelab/client/tui/app.py

from dishka import make_async_container, AsyncContainer
from codelab.client.infrastructure.app_provider import AppProvider


class ACPClientApp(App):
    def __init__(self, *, host: str, port: int, cwd: str, container: AsyncContainer):
        super().__init__()
        self._host = host
        self._port = port
        self._cwd = cwd
        self._container = container  # владение у caller-а, не у App

    async def on_mount(self) -> None:
        self._ui_vm = await self._container.get(UIViewModel)
        self._session_vm = await self._container.get(SessionViewModel)
        self._chat_vm = await self._container.get(ChatViewModel)
        # ... остальные ViewModels

    async def on_unmount(self) -> None:
        pass  # cleanup делает `async with` в _run_tui_app_async


async def _run_tui_app_async(*, host: str, port: int, cwd: str | None, history_dir: str | None) -> None:
    resolved_cwd = os.getcwd() if cwd is None else os.path.abspath(os.path.expanduser(cwd))
    if not os.path.exists(resolved_cwd) or not os.path.isdir(resolved_cwd):
        raise ValueError(f"Путь {resolved_cwd} не является доступной директорией")

    provider = AppProvider(host=host, port=port, cwd=resolved_cwd, history_dir=history_dir)
    async with make_async_container(provider) as container:
        app = ACPClientApp(host=host, port=port, cwd=resolved_cwd, container=container)
        await app.run_async()
    # ← container.close() вызывается здесь гарантированно, даже при исключении


def run_tui_app(*, host=None, port=None, cwd=None, history_dir=None) -> None:
    resolved_host, resolved_port = resolve_tui_connection(host=host, port=port)
    asyncio.run(
        _run_tui_app_async(host=resolved_host, port=resolved_port, cwd=cwd, history_dir=history_dir)
    )
```
