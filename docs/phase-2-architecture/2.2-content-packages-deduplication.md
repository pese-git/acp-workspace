# 2.2 — Устранение дублирования пакетов `content`

**Приоритет:** 🟠 Высокий  
**Оценка:** 1 день  
**Файлы:** `src/codelab/server/protocol/content/`, `src/codelab/shared/content/`

---

## Проблема

В проекте два пакета с пересекающимися файлами:

| `server/protocol/content/` | `shared/content/` | Статус |
|---|---|---|
| `base.py` | `base.py` | 🔴 Дублирование |
| `audio.py` | `audio.py` | 🔴 Дублирование |
| `embedded.py` | `embedded.py` | 🔴 Дублирование |
| `image.py` | `image.py` | 🔴 Дублирование |
| `text.py` | `text.py` | 🔴 Дублирование |
| `resource_link.py` | `resource_link.py` | 🔴 Дублирование |
| `extractor.py` | *(нет)* | ✅ Только в server |
| `formatter.py` | *(нет)* | ✅ Только в server |
| `validator.py` | *(нет)* | ✅ Только в server |

При изменении бизнес-логики в одном месте второе место останется устаревшим.

---

## Стратегия миграции

1. Оставить дублирующиеся модели в `shared/content/` — это единственный источник правды.
2. Переместить `extractor.py`, `formatter.py`, `validator.py` в `server/protocol/content/`, при этом они импортируют из `shared/content/`.
3. Удалить дублирующиеся файлы из `server/protocol/content/`.
4. Обновить все импорты.

---

## Шаг 1 — Сравнить содержимое дублирующихся файлов

Перед удалением убедиться, что `server/protocol/content/` не содержит расширений, которых нет в `shared/content/`:

```bash
# Сравнить каждую пару:
diff src/codelab/server/protocol/content/base.py src/codelab/shared/content/base.py
diff src/codelab/server/protocol/content/audio.py src/codelab/shared/content/audio.py
diff src/codelab/server/protocol/content/text.py src/codelab/shared/content/text.py
# ... и т.д.
```

Если в `server/protocol/content/base.py` есть что-то, чего нет в `shared/content/base.py` — добавить в `shared/content/base.py` перед удалением.

### Шаг 2 — Обновить `shared/content/__init__.py`

Убедиться, что `shared/content/` экспортирует всё необходимое:

```python
# src/codelab/shared/content/__init__.py
from .base import ContentBase, ContentType
from .text import TextContent
from .image import ImageContent
from .audio import AudioContent
from .embedded import EmbeddedContent
from .resource_link import ResourceLinkContent

__all__ = [
    "ContentBase",
    "ContentType",
    "TextContent",
    "ImageContent",
    "AudioContent",
    "EmbeddedContent",
    "ResourceLinkContent",
]
```

### Шаг 3 — Обновить `server/protocol/content/` — оставить только специфичное

```python
# src/codelab/server/protocol/content/__init__.py — ПОСЛЕ рефакторинга:
# Реэкспортируем из shared для обратной совместимости внутренних импортов
from codelab.shared.content import (
    ContentBase,
    ContentType,
    TextContent,
    ImageContent,
    AudioContent,
    EmbeddedContent,
    ResourceLinkContent,
)

# Специфичные для сервера утилиты остаются здесь:
from .extractor import ContentExtractor
from .formatter import ContentFormatter
from .validator import ContentValidator

__all__ = [
    # из shared:
    "ContentBase", "ContentType", "TextContent", "ImageContent",
    "AudioContent", "EmbeddedContent", "ResourceLinkContent",
    # специфичные для сервера:
    "ContentExtractor", "ContentFormatter", "ContentValidator",
]
```

### Шаг 4 — Удалить дублирующиеся файлы

```bash
cd src/codelab/server/protocol/content/

# Удалить файлы, которые теперь живут в shared/content/:
rm base.py audio.py embedded.py image.py text.py resource_link.py
```

### Шаг 5 — Обновить `extractor.py`, `formatter.py`, `validator.py`

Заменить импорты в оставшихся файлах:

```python
# БЫЛО (в extractor.py):
from .base import ContentBase
from .text import TextContent
from .image import ImageContent

# СТАЛО:
from codelab.shared.content import ContentBase, TextContent, ImageContent
```

### Шаг 6 — Обновить все импорты в проекте

```bash
# Найти все файлы, импортирующие из server/protocol/content:
grep -rn "from.*server\.protocol\.content\|from.*protocol\.content" \
  src/codelab/ --include="*.py" | grep -v "extractor\|formatter\|validator"
```

Для каждого найденного файла — заменить импорт:

```python
# БЫЛО:
from codelab.server.protocol.content.text import TextContent
from codelab.server.protocol.content.base import ContentBase

# СТАЛО:
from codelab.shared.content import TextContent, ContentBase
```

### Шаг 7 — Обновить тесты

```bash
# Найти тесты, дублирующиеся аналогично модулям:
ls tests/server/test_content_*.py
ls tests/client/test_content_*.py

# Объединить в tests/shared/test_content_*.py или удалить дубликаты,
# оставив тесты в одном месте:
mkdir -p tests/shared
mv tests/server/test_content_base.py tests/shared/
# Удалить tests/client/test_content_base.py если содержание идентично
```
